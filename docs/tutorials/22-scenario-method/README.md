# 第 22 章: 场景设计题万能答题方法论

> **核心问题**：面试中遇到"设计一个 XXXX Agent"，怎么在 15 分钟内给出结构化回答？

## 四步答题框架

### Step 1: 需求拆解（2 分钟）

先搞清楚"到底要设计什么"：

| 问题 | 目的 |
|------|------|
| 这个 Agent 的主要用户是谁？ | 确定体验要求 |
| 最核心的功能是什么？ | 确定 MVP |
| 有哪些非功能约束？ | 延迟、安全、成本 |
| 数据来源是什么？ | 确定 RAG/知识库策略 |

### Step 2: 架构选型（3 分钟）

画出架构图，选择关键组件：

```
用户输入 → 路由 → 简单任务 → 小模型 + 缓存
                → 复杂任务 → Plan Mode + RAG + 工具
                → 敏感任务 → 人工审批 + 日志全追踪
```

| 组件 | 选项 | 选型依据 |
|------|------|---------|
| 主模型 | GPT-4 / Claude / 本地模型 | 成本 vs 质量 |
| 框架 | 自研 / LangChain / Codex | 灵活性 vs 开发效率 |
| 记忆 | Context Window / 向量库 | 会话长度 |
| 知识库 | RAG / Fine-tuning / 纯 Prompt | 知识更新频率 |

### Step 3: 核心模块设计（5 分钟）

细化 3-4 个关键模块，说明输入、处理、输出：

1. **意图识别模块**：分类器（LLM 或小模型）→ 路由到不同处理链
2. **RAG 检索模块**：Query rewriting → 向量检索 → 重排 → 注入
3. **工具执行模块**：Schema 定义 → 参数校验 → 执行 → 结果格式化
4. **安全防护模块**：输入过滤 → Prompt 注入检测 → 输出校验

### Step 4: 风险与优化（5 分钟）

最后展示你的工程思维：

| 风险 | 应对 |
|------|------|
| 上下文溢出 | 滑动窗口 + 摘要压缩 |
| LLM 幻觉 | 知识库校验 + Judge 模型 |
| 高延迟 | 缓存 + 并行调用 + 小模型预处理 |
| 成本超支 | 模型分层 + Token 预算控制 |
| 安全风险 | Prompt 注入检测 + 人工审批兜底 |

## 两个实战案例

### 案例：设计客服 Agent

```
需求：客服机器人，处理 80% 常见问题，复杂问题转人工
架构：意图识别 → 简单 FAQ（缓存命中直接回复）→ RAG 问答 → 转人工
亮点：情绪检测触发转人工、未解决问题自动创建工单
```

### 案例：设计代码审查 Agent

```
需求：PR 自动审查，发现 Bug 和安全隐患
架构：Diff 分析 → 静态检查（规则引擎）→ LLM 语义审查 → 汇总报告
亮点：误报率追踪、增量审查（只审查变更部分）
```
