# 第 21 章: 高频面试题汇总与标准答案

> 覆盖 Agent 开发岗位的四大类高频面试题。

## 基础概念类

**Q1: 什么是 Agent？和 LLM 有什么区别？**
Agent 是基于 LLM 的自主系统，能感知环境、做出决策、执行行动。LLM 只是一个文本生成模型，Agent 在 LLM 基础上增加了循环、工具调用、记忆等能力。

**Q2: 什么是 ReAct 模式？**
ReAct = Reasoning + Acting。Agent 交替进行"思考（推理）→ 行动（工具调用）→ 观察（结果收集）"的循环过程。先思考要做什么，再行动，然后根据结果继续思考。

**Q3: 什么是 Plan Mode？和 ReAct 有什么区别？**
Plan Mode 是"先规划再执行"的模式：Agent 先制定完整计划，然后逐步执行。ReAct 是"边想边做"的模式。Plan Mode 适合复杂任务（减少试错成本），ReAct 适合简单任务（响应更快）。

**Q4: Token 是什么？怎么估算成本？**
Token 是 LLM 处理文本的最小单位。英文约 1 token/0.75 单词，中文约 1-2 token/字。成本 = Prompt Tokens × 单价 + 输出 Tokens × 单价。

## 架构原理类

**Q5: Agent 的核心执行循环是怎么工作的？**
Think（LLM 推理生成下一步 Action）→ Act（执行工具调用）→ Observe（收集结果）→ 回到 Think。循环直到满足终止条件（任务完成、达到最大轮次、遇到错误）。

**Q6: 怎么设计 Memory 系统？**
两级存储：短期记忆（上下文窗口中的最近对话）+ 长期记忆（外部存储：向量数据库/JSON 文件）。关键设计点：FIFO 淘汰机制、信息提取与压缩、记忆检索与注入。

**Q7: 什么是 RAG？为什么 Agent 需要 RAG？**
RAG = Retrieval-Augmented Generation。先检索相关知识，再让 LLM 基于检索到的内容生成回答。Agent 需要 RAG 来解决 LLM 的知识截止问题和幻觉问题。

**Q8: Multi-agent 系统有哪些架构模式？**
三种：层级式（Manager + Workers）、协作式（Peer-to-peer 讨论）、对抗式（Generator + Critic 互相校验）。选择依据：任务复杂度、容错需求、通信开销。

**Q9: Harness 是什么？**
Harness 是 Agent 的执行框架，负责管理 LLM 调用、工具调度、循环控制、上下文维护。相当于 Agent 的"操作系统"。

## 工程落地类

**Q10: Agent 的上下文溢出怎么解决？**
滑动窗口（保留最近 N 轮对话）、摘要压缩（LLM 将历史压缩为摘要）、Memory 系统（关键信息外存）。实际生产中是组合使用。

**Q11: Agent 的降级策略有哪些？**
四级降级：L1 工具降级（主→备→降级回复）、L2 模型降级（强→弱→规则）、L3 功能降级（完整→只读→提示）、L4 人工兜底。

**Q12: 怎么评估 Agent 的效果？**
五维评估：完成率、准确率、幻觉率、轮次、耗时。方法：自动化测试 + LLM-as-Judge + 人工抽样。关键：建立基线做回归对比。

**Q13: 怎么降低 Agent 的 Token 成本？**
上下文裁剪、摘要压缩、语义缓存、模型分层（简单任务用小模型）、流式输出。

**Q14: Agent 服务化有哪些关键设计？**
API 封装（REST + 流式）、状态管理（Redis 持久化）、可靠性（限流 + 降级 + 超时）、多租户隔离（上下文/数据/速率隔离）。

## 面试技巧

**回答结构**：先给结论，再展开细节，最后说权衡。例如："这个问题核心是 XXX。具体来说有三点...。实际应用中需要注意...。"
