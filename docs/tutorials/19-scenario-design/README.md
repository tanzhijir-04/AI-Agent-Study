# 第 19 章: 典型业务场景 Agent 设计

> **核心问题**：面试中让你"设计一个客服/代码/数据分析 Agent"，应该怎么构思？

## 场景 1：智能客服 Agent

### 需求拆解

| 需求 | 说明 |
|------|------|
| **核心功能** | 自动回答用户问题、转接人工、工单创建 |
| **知识来源** | 产品文档、FAQ 库、历史工单 |
| **约束条件** | 响应 < 3s、不能捏造信息、敏感问题转人工 |

### 架构选型

```
用户 → 意图识别 → 常规问题 → RAG 检索 → 生成回复
                → 投诉/退款 → 转人工客服
                → 查询订单 → 调用订单 API → 生成回复
```

### 关键设计

| 组件 | 选型 | 理由 |
|------|------|------|
| 主模型 | GPT-4o-mini | 性价比高，多数问答不需要最强模型 |
| RAG 方案 | 向量库 + 关键词混合检索 | 提高召回率 |
| 转人工条件 | 用户情绪检测、3 轮未解决、敏感词触发 | 防止机器人激怒用户 |
| 安全防护 | Prompt 注入检测 + 输出合规校验 | 防止恶意利用 |

## 场景 2：代码助手 Agent

### 核心能力

1. **代码生成**：根据描述生成代码片段
2. **代码审查**：Review PR，发现潜在问题
3. **重构建议**：识别代码异味，给出改进方案
4. **文档生成**：为代码生成注释和文档

### 架构特点

```javascript
// 代码助手需要理解代码上下文
class CodeAssistant {
  constructor() {
    this.context = new CodeContext(); // 当前文件 + 相关文件
    this.linter = new Linter();       // 静态分析
    this.tester = new TestRunner();   // 运行测试验证生成代码
  }

  async generateCode(request) {
    // 1. 收集上下文
    const ctx = await this.context.gather();

    // 2. 生成代码
    const code = await llm.generate(request, ctx);

    // 3. 静态检查
    const issues = this.linter.check(code);
    if (issues.length > 0) {
      return this.fixIssues(code, issues);
    }

    return code;
  }
}
```

## 场景 3：数据分析 Agent

### 工作流程

```
用户提问（自然语言）
    ↓
Agent 理解意图 → 生成 SQL/代码
    ↓
执行查询（沙箱环境）
    ↓
分析结果
    ↓
生成可视化（图表）
    ↓
自然语言解释
```

### 安全要点

- **只读数据库**：防止 Agent 修改数据
- **查询超时**：复杂查询限制执行时间
- **结果采样**：大数据集先采样再分析
- **代码沙箱**：所有生成的代码在隔离环境运行

## 场景设计通用框架

回答任何场景设计题的"四步框架"：

```
1. 需求拆解
   明确功能需求 + 非功能约束（延迟、安全、成本）

2. 架构选型
   选择合适的模型、工具、存储方案

3. 核心模块设计
   细化每个组件的实现要点

4. 风险与优化
   识别风险点并给出应对方案
```

## 面试高频问法

**Q1: 设计一个智能客服 Agent，你会怎么选型？**
按复杂度分层：简单问答用 RAG + GPT-4o-mini、复杂推理用 GPT-4、投诉情绪检测用独立模型。转人工策略是必考点。

**Q2: 知识库问答 Agent 和普通 RAG 有什么区别？**
知识库 Agent 需要多轮对话能力：根据上下文追问、澄清模糊问题、区分"不知道"和"没有这个知识"。普通 RAG 只是单轮检索 + 生成。

**Q3: DevOps Agent 怎么处理告警风暴？**
核心是"去重 + 降噪 + 关联分析"：先聚合相似告警、再按优先级排序、然后分析告警之间的关联关系（同一个根因触发的多个告警合并处理）。
